RuntimeType concept

The RuntimeType concept provides at runtime data and functionalities specific to a type (the target type), like ctors, dtor, or retrieval of size and alignment.

The target type is assigned with the make static function template (see below). A default constructed RuntimeType is empty, that is it has no target type. Trying to use any feature of the target type on an empty RuntimeType results is undefined behavior.

Requirement Semantic
Non-throwing default constructor and destructor A default constructed RuntimeType is empty (not assigned to a target type).
Copy constructor and copy assignment The destination RuntimeType gets the same target type of the source RuntimeType.
Non-throwing move constructor and non-throwing move assignment The destination RuntimeType gets the target type of the source RuntimeType. The source becomes empty.
Operators == and != Checks for equality\inequality. Two RuntimeType are equal if they have the same target type.
Type alias:
using common_type = [implementation defined]
The type to which all the types handled by this RuntimeType are covariant. common_type can be void, is which case any target type is legal.
